## Исправления

- **Свежие сообщения больше не теряются за демо-данными** — Список сообщений [`GET /api/messages`](/docs/ru/api-reference/messages/list) и поиск [`GET /api/messages/search`](/docs/ru/api-reference/messages/search) теперь сортируют результаты по дате сообщения (сначала новые), а не по `id`. Идентификаторы сообщений Telegram считаются отдельно для каждого чата и не являются глобально монотонными, поэтому только что отправленные сообщения с низкими ID могли вытесняться более старыми сообщениями с высокими ID при достижении лимита. Та же сортировка применяется и к «последнему сообщению» в списке чатов на дашборде.
