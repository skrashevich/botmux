## Новые функции

- **Прокси для загрузки файлов** — BotMux теперь предоставляет эндпоинт для загрузки файлов по адресу `/tgapi/file/bot{TOKEN}/{file_path}`, который работает как полная замена URL загрузки файлов Telegram. Вы можете настроить его как `base_file_url` в вашей библиотеке для ботов, и BotMux будет передавать файлы напрямую из Telegram. Стикеры в формате WebP автоматически конвертируются в PNG для совместимости с браузерами, а ответы включают правильные заголовки кэширования. Подробности в разделе [API-прокси](/docs/ru/features/api-proxy).

## Улучшения

- **Расширенный перехват методов** — [API-прокси](/docs/ru/features/api-proxy) теперь перехватывает вызовы `logOut` и `close` Telegram API в дополнение к методам вебхуков и опроса. Вызовы `logOut` блокируются для предотвращения отключения токена бота на 10 минут, а вызовы `close` блокируются для предотвращения остановки экземпляра бота на стороне Telegram.

- **Улучшенная обработка getUpdates** — Когда бот управляется BotMux, но длинный опрос не включен, API-прокси теперь возвращает пустой результат для `getUpdates` вместо пересылки запроса в Telegram. Это предотвращает ошибки конфликта, когда ваш бэкенд вызывает `getUpdates`, пока BotMux уже опрашивает того же бота. Подробнее о [длинном опросе](/docs/ru/features/long-polling).

- **Документация на китайском языке** — Документация BotMux теперь доступна на китайском (упрощённом) языке наряду с английским и русским. Переключайте язык в панели навигации. Подробнее о [поддержке языков](/docs/ru/features/i18n).
