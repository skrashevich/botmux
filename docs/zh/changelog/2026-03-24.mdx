## 新功能

- **用户资料** — 点击仪表板中的任何用户名即可查看资料弹窗，显示用户统计数据和最近操作。用户名现在在消息视图和用户列表中均可点击。

- **更新通知** — BotMux 现在会检查 GitHub 上的新版本，并在有更新时显示可关闭的横幅。您也可以从仪表板页脚手动检查更新。了解更多关于[部署](/docs/zh/deployment/requirements)的信息。

- **Webhook 拦截** — BotMux 现在会拦截通过 [API 代理](/docs/zh/features/api-proxy)发出的 `setWebhook`、`deleteWebhook` 和 `getWebhookInfo` API 调用。这意味着当您的后端设置 Webhook 时，机器人会自动配置，无需手动设置。

- **新品牌标识** — BotMux 推出了新的 Hub 标志，体现其多路复用器功能。标志显示在侧边栏、登录界面和浏览器收藏夹图标中。

- **外部化翻译** — 界面翻译现在从外部文件加载，而不是内联捆绑。这使得贡献新翻译更加容易，并保持界面代码更整洁。英语和俄语开箱即用。详情请参阅 [i18n](/docs/zh/features/i18n)。

## 改进

- **页脚版本显示** — 仪表板页脚现在显示当前 BotMux 版本，并提供手动检查更新的按钮。

- **增强的日志查看器** — [应用日志查看器](/docs/zh/features/analytics)现在更大，在日志条目旁显示机器人名称，并使用改进的颜色编码以便于浏览。

- **改进的长轮询** — 改进了[长轮询](/docs/zh/features/long-polling)更新的参数解析和处理。界面现在显示更清晰的长轮询配置提示，当机器人启用长轮询时，后端转发会自动跳过。

- **改进的媒体下载** — Telegram [文件代理](/docs/zh/features/media-support)现在更可靠地处理媒体下载，改进了文件检索和内容传递。

## 错误修复

- **修复了高负载下的数据库锁定** — 为 SQLite 连接添加了繁忙超时，防止多个请求同时访问数据库时出现锁定错误。

- **修复了重复轮询** — BotMux 现在会阻止对同一机器人的并发 `getUpdates` 调用，防止代理和长轮询客户端同时尝试获取更新时产生冲突。

- **修复了长轮询参数解析** — 切换为结构化 JSON 解析长轮询请求参数，解决了更新偏移量或限制未正确应用的边缘情况。
