## 新功能

- **机器人禁用/启用** — 您现在可以在仪表板中临时禁用机器人而无需删除它。禁用的机器人停止轮询和代理，但保留所有配置和历史记录。随时一键重新启用。详情请参阅[多机器人管理](/docs/zh/concepts/multi-bot)。

- **回调查询处理** — BotMux 现在处理内联按钮交互（回调查询）并将回调数据存储在消息历史中。这让您可以完整了解用户如何与机器人的内联键盘进行交互。

- **编辑消息跟踪** — [API 代理](/docs/zh/features/api-proxy)现在除了发送方法外，还拦截 `editMessageCaption` 和 `editMessageMedia`。编辑后的消息会被捕获并在仪表板中实时更新，因此消息历史始终反映最新内容。

## 改进

- **Telegram Bot API 9.6 兼容性** — BotMux 现在支持最新的 Telegram Bot API 更新类型，包括商业消息和加入聊天请求。机器人发送的消息在仪表板中以徽章进行视觉区分，并在可用时显示发送者标签。

- **自定义 API 端点支持** — 机器人现在可以配置自定义 Telegram API 基础 URL，使 BotMux 更容易与自托管的 [Telegram Bot API 服务器](/docs/zh/configuration/custom-telegram-api)配合使用。

## 错误修复

- **修复仪表板中的 XSS 漏洞** — 按钮处理程序中的用户输入现在已正确转义，防止潜在的跨站脚本攻击。

- **修复聊天列表显示错误的消息预览** — 解决了当多个机器人共享同一聊天时，聊天列表可能显示来自其他机器人的消息预览的问题。
